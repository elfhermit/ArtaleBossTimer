# ArtaleBossTimer

一個簡單好上手的 Boss 擊殺紀錄工具，幫助你記錄每次擊殺、追蹤已出貨狀態，並自動預估下一次復活時間。

適合對象

- 遊戲玩家、公會管理者或任何需要追蹤 Boss 出現/擊殺時間的人。

主要功能

- 快速記錄：選擇 Boss、輸入頻道、標註是否出貨與備註，立即儲存擊殺紀錄。
- 預估復活時間：系統會根據既有規則自動計算並顯示該 Boss 的下次預估復活時間，方便你安排狩獵或競賽。
- 智慧狀態提示：復活時間欄位會以顏色標示狀態
  - 🟧 **橙色**：復活時間範圍內（Boss 隨時可能出現）
  - 🟩 **綠色**：已超過復活時間（Boss 必定已出現）
  - 無特殊標示：復活時間尚未到達
- 24 小時制時間：所有時間以 24 小時制顯示（例如：14:05），避免 AM/PM 混淆。
- 可排序紀錄列表：點擊表頭可依時間、頻道、出貨狀態、備註或復活時間排序，方便快速找到需要的紀錄。
- 當日紀錄檢視：清楚列出當天的所有紀錄，支援快速搜尋、排序與過濾。
- 編輯與刪除：可直接在紀錄列表中修改或移除錯誤紀錄，支援撤銷刪除操作。
- 匯入 / 匯出備份：提供備份與還原資料的機制，讓你能保存重要紀錄或在不同裝置間移轉（資料僅存在於你的裝置上）。

為什麼使用它？

- 省時：快速輸入與自動計算，減少手動整理紀錄的時間。
- 清晰：一目了然的列表與預估時間，方便你規劃行程與協調隊伍。
- 私密：所有紀錄都保存在你的裝置中，不會自動上傳到外部伺服器（請依照你的需求定期備份）。

快速上手（給一般使用者）

1. 開啟應用程式：在瀏覽器中開啟 `docs/index.html`（或將整個專案以靜態伺服器提供）。
2. 選擇 Boss：點選左側任一 Boss 卡片來選擇要記錄的首領。
3. 填寫紀錄：在右側的「紀錄擊殺」表單中設定擊殺時間、頻道（1-3000）、是否出貨與備註，點選「確認新增紀錄」。
4. 查看歷史：下方「近期擊殺歷史紀錄」會列出所有紀錄，支援刪除單筆或清除全部紀錄。
5. 預估復活時間：系統會根據該 Boss 的復活規則計算「最早／最晚」復活時間，並在卡片顯示狀態與進度條。

主要畫面說明

- 左側：即時戰況監控（Boss 卡片） — 顯示每個 Boss 的復活視窗、狀態與倒數計時。
- 右側：紀錄擊殺（側邊欄） — 用於新增擊殺紀錄的表單與快速頻道選擇。
- 下方：近期擊殺歷史紀錄（表格） — 顯示擊殺時間、頻道、是否出貨、備註與預計復活區間。

資料儲存與隱私

- 本應用採用瀏覽器的 LocalStorage 儲存使用者資料，資料存放在你的裝置上，不會自動上傳至外部伺服器。
- LocalStorage 的 key：目前程式使用 `bossKillHistory`（請參閱 `docs/app.js` 中的 `loadHistory` / `saveHistory`）。
- 建議定期匯出備份（手動複製 LocalStorage 的 JSON），以免裝置資料遺失。

本機啟動（開發 / 本地預覽）

1. 使用 Python 內建 HTTP 伺服器（在專案根目錄執行）：

```powershell
python -m http.server 8000
# 然後在瀏覽器開啟 http://localhost:8000/docs/index.html
```

2. 或者使用其他靜態伺服器工具（如 `live-server`、`http-server` 等），只要能提供 `docs/` 目錄給瀏覽器即可。

進階說明（短摘要）

- 規則來源：復活規則與資料模型可在 `docs/bosses/bosses.json` 與 `dev/SPEC.md`（若存在）中查看。
- 時間處理：系統使用 ISO 時間與本地時間輸入（`datetime-local`），所有時間以 24 小時制顯示。
- 篩選與提示：卡片支援依狀態篩選（全部 / 已存活 / 即將重生 / 冷卻中），復活時間會以顏色與進度條提示。

常見問題

- 我可以在手機上使用嗎？

   是的，介面為響應式，可在多數現代瀏覽器的手機或平板上運行。

- 資料會不會被分享到網路？

   所有紀錄預設儲存在你的裝置上；若你要備份或分享，請使用匯出功能。
