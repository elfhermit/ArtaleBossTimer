# Artale Boss Timer 開發文件

## 需求
我想要建立一個靜態頁面放置於 GitHub Pages 上。
其內容用於紀錄我殺死 Boss 的紀錄，以及我想知道我何時在何頻道擊殺了哪個 Boss，並且是否有出貨。
後面也需要統計該Boss 當天的所有擊殺紀錄，以及同個頻道的Boss 可能復活時間。

目前需求如下:

由於 User 通常會針對 同個Boss 多次擊殺，會刷新頻道來去找Boss 並擊殺，因此對於同個 Boss 會有多筆擊殺紀錄。

網頁版面需要控管，未來可以針對某 Boss 去找當天歷史紀錄 頁面保留 3000 筆擊殺紀錄。(紀錄於該 Client瀏覽器 的 Local Storage)
** 當天以 00:00:00 ~ 23:59:59 為一天 **

需要紀錄的欄位
``` 同一筆擊殺紀錄
- Boss 名稱 (自動帶入，因為前一個動作會選擇今天想殺的Boss)
- 擊殺日期 需要到 hh-MM-ss (時-分-秒) (填完資料自動帶入當下時間)
- 頻道名稱(數字) 1~3000 (使用者輸入, 判斷為正整數)
- 是否出貨 (是/否) (radio button)
- 備註 (可選) (text area)
```

### 功能需求
- 使用者可以選擇今天想要擊殺的 Boss (下拉選單)
- 使用者可以填寫擊殺紀錄 (表單)
- 使用者可以查看當天針對某 Boss 的所有擊殺紀錄 (表格)
- 使用者可以編輯或刪除已存在的擊殺紀錄

- 需要提供 Boss 復活時間的參考資訊 (靜態資料)
  - Boss 復活時間為固定時間 (例如: 每小時的 15 分鐘後復活 or 一個範圍時間 如 45~60 分鐘後復活)
  - 提供一個簡單的計算器，讓使用者輸入擊殺時間和頻道，計算出該頻道的 Boss 可能復活時間
  
### 技術需求
- 使用 GitHub Pages 來部署靜態頁面
- 使用 HTML/CSS/JavaScript 來實現前端功能
- 使用 Local Storage 來儲存擊殺紀錄
- 頁面需具備響應式設計，適應不同裝置瀏覽
- 程式碼需具備良好結構與註解，方便未來維護與擴充
- 滿足 Clean Code 原則，確保程式碼易讀且易於理解
