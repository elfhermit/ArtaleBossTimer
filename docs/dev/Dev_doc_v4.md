# Boss 監控儀表板 (Boss Timer Dashboard) - UI/UX 優化建議書

**文件日期**: 2025/11/17
**設計目標**: 從「資料庫查詢工具」轉型為「即時戰情儀表板」，提升玩家在遊戲進行中的操作效率與資訊判讀速度。

## 1\. 現狀體驗分析 (UX Audit)

根據目前的介面截圖，發現以下使用痛點：

  * **資訊密度低**：左側列表僅作為導航，未顯示動態資訊（如剩餘時間），導致使用者必須頻繁點擊才能查看狀態。
  * **視線斷層 (Eye Travel)**：使用者需在左側「選擇 Boss」與右側「輸入資料」之間大幅度移動視線。
  * **操作路徑長**：紀錄一次擊殺需經過「搜尋/捲動 -\> 選擇 -\> 移動滑鼠 -\> 輸入時間 -\> 送出」多個步驟。

-----

## 2\. 版面佈局重構 (Layout Redesign)

建議採用 **70/30 分割佈局**，將「監控」與「操作」緊密結合。

### 2.1 左側：戰況監控區 (70% 寬度)

  * **定位**：主視覺區域，取代原本窄小的列表。
  * **形式**：寬版響應式表格 (Responsive Table) 或 卡片清單。
  * **排序邏輯**：預設依 `狀態` 排序（已重生 \> 即將重生 \> 冷卻中），讓玩家一眼看到「現在能打誰」。

### 2.2 右側：快速作業區 (30% 寬度)

  * **定位**：輔助操作區域。
  * **行為**：設定為 **Sticky (固定懸浮)**，當左側表格過長捲動時，右側輸入框永遠保持在視窗可見範圍內。
  * **連動性**：點擊左側任一 Boss，右側表單自動帶入該 Boss 資料與當下時間。

-----

## 3\. 介面功能細節 (Feature Specifications)

### A. 智慧監控表格 (The Smart Table)

表格應包含以下欄位與視覺提示：

| 欄位 | 顯示內容建議 | 視覺設計 (UI Hint) |
| :--- | :--- | :--- |
| **Boss** | 圖片 + 名稱 | 圖片增加辨識度，滑鼠 Hover 時顯示詳細重生區間。 |
| **狀態** | 倒數計時 / 存活 | **綠色燈號**：已重生 (Alive)<br>**黃色燈號**：\< 5分鐘 (Warning)<br>**紅色燈號**：冷卻中 (Dead) |
| **倒數條** | 進度條 (Progress Bar) | 針對冷卻中的 Boss，顯示視覺化進度條，直觀感受剩餘時間。 |
| **上次擊殺** | 時間 + 頻道 | 例如 `13:30 (Ch.5)`，方便追查。 |
| **操作** | 快速按鈕 | 直接顯示 `[⚔️ 紀錄擊殺]` 按鈕，點擊後直接觸發右側表單或 Modal。 |

### B. 極速輸入表單 (The Quick Entry Form)

針對遊戲情境優化輸入體驗：

1.  **預設值自動化**：
      * **時間**：預設帶入 `new Date()` (當前時間)，並提供 `±1分` 微調按鈕。
      * **Boss**：由左側點擊觸發帶入。
2.  **頻道選擇器 (Channel Grid)**：
      * **痛點**：下拉選單 (Dropdown) 操作慢。
      * **解法**：改用 **Grid Button Group** (如 1\~20 的數字方塊)，點一下即選中。
3.  **戰利品開關**：
      * 使用 Toggle Switch (開關元件) 取代 Radio Button，操作感更明確。

## 5\. 給開發者的實作建議 (Technical Notes)

若您使用現代前端框架 (React/Vue/Angular)，以下是實作建議：

  * **State Management**:
      * 建議將 `SelectionState` (當前選中的 Boss) 提升到父層，讓左側 Table 與右側 Form 共享狀態。
  * **CSS Styling**:
      * 右側 Sidebar 使用 `position: sticky; top: 20px;` 確保捲動時的可視性。
      * 狀態顏色建議使用標準語意色：
          * Success: `#4CAF50` (綠 - 可擊殺)
          * Warning: `#FFC107` (黃 - 快出了)
          * Danger/Muted: `#F44336` 或 `#9E9E9E` (紅/灰 - 沒戲唱)
  * **Clipboard API**:
      * 建議增加「複製公會公告」功能。點擊按鈕後，將當前資料格式化為文字串：
        > `紅寶王(Ch5) 已出 / 蘑菇王 剩3分 / 巴洛古 14:00 出`
        > 方便玩家直接貼到 Discord 或 Line 群組。
