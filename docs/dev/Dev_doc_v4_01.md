# Boss 監控儀表板 (Boss Timer Dashboard) - UI/UX 優化建議書

這套設計使用了 Material Design 風格，整體的質感提升了很多。針對你提供的截圖，我做了一份 Design Review，指出**做得好的地方**以及**需要急救的關鍵點**。

-----

### 🕵️‍♂️ Design Review：UI/UX 檢視報告

#### ✅ 成功達成的項目 (Good Job)

1.  **暗黑模式 (Dark Mode)**：非常適合遊戲玩家，長時間觀看舒適度大幅提升。
2.  **即時戰況監控 (Card View)**：左側改用卡片+進度條，資訊層次比純文字好太多了。
3.  **版面權重 (Layout)**：左右分割 (Main/Sidebar) 的邏輯正確，輸入區在右側很直覺。
4.  **操作優化**：加入了「是否出貨」的 Toggle Switch，以及卡片上的「紀錄擊殺」按鈕，這些都減少了認知負擔。

#### ⚠️ 建議修正的關鍵點 (Critical Fixes)

雖然架構對了，但目前有幾個**版面擁擠**與**邏輯重複**的問題，建議優先處理：

**1. 右下角的「紀錄清單」崩壞了 (Layout Break)**

  * **問題**：目前的紀錄清單被擠在右側狹窄的空間，導致表頭（首領、時間...）變成垂直排列，內容完全跑版，難以閱讀。
  * **解法**：**請把「紀錄清單」移回左側，放在「即時戰況監控」的下方。**
      * 表格 (Table) 天生需要寬度，右側欄位 (Sidebar) 只適合放「輸入表單」或「簡單摘要」。
      * 移到左側下方後，可以做成寬版表格，閱讀性會瞬間滿分。

**2. 左側出現了兩份列表 (Redundancy)**

  * **問題**：上方有「即時戰況監控 (卡片)」，下方又有「快速篩選 / Boss 列表 (表格)」。這讓使用者困惑：「我到底要看哪邊？」且佔用了寶貴的垂直空間。
  * **解法**：**移除下方的「快速篩選列表」**。
      * 上方的「卡片區」就是你的列表。
      * 如果需要篩選，請在卡片區上方加一個 Search Bar 或 Filter Chips (例如：[全部] [存活中] [即將重生]) 即可。

**3. 進度條缺乏「語意色」 (Color Meaning)**

  * **問題**：目前的進度條都是白色的。玩家無法用眼角餘光判斷誰是紅血（快生了）誰是滿血（剛死）。
  * **解法**：利用 Material Design 的色彩系統：
      * **剛死 (冷卻中)**：灰色或半透明白色 (Disabled/Surface Variant)。
      * **快重生 (例如剩 10 分鐘)**：黃色/琥珀色 (Warning/Secondary)。
      * **已重生**：將整張卡片邊框亮起，或是按鈕變為顯眼的綠色 (Success/Primary)。

### 🎨 針對 Material Design 的具體調整建議

既然你使用 Material Design + 標準 CSS，這裡有一些元件建議，不用 Tailwind 也能輕鬆刻出來：

#### 1\. 頻道選擇 (Channel Input)

目前右側看起來還是 Text Input。建議改用 **Choice Chips** 風格。

  * **CSS 概念**：
    ```css
    .channel-grid {
      display: grid;
      grid-template-columns: repeat(5, 1fr); /* 一排5個 */
      gap: 8px;
    }
    .channel-chip {
      border: 1px solid #555;
      border-radius: 16px; /* 圓角 */
      text-align: center;
      padding: 4px 0;
      cursor: pointer;
    }
    .channel-chip.active {
      background-color: #bb86fc; /* Material Purple 或你的主色 */
      color: #000;
      border-color: #bb86fc;
    }
    ```

### ✅ 已實作變更（範例實作）

以下為根據上方建議所做的小幅前端實作，已直接套用於 `docs/` 靜態頁面（2025-11-17）：

- **紀錄清單移置**：將原本在右側的 `紀錄清單` (`#records-root`) 移到左側，放在 `即時戰況監控` 下方，表格現在可使用較寬的空間呈現。
- **快速列表重定位**：原先左側的 `快速篩選 / Boss 列表` (`#boss-list-compact` / `#boss-dropdown`) 轉移到右側側邊欄（Sidebar）以避免冗餘資訊；右側保留 `新增紀錄` 表單。
- **進度條語意色**：進度條支援三種語意狀態（冷卻 gray / 復活中 amber / 已復活 green），UI 會根據計算出的 respawn 狀態為卡片加上對應 class (`respawn-cooling`, `respawn-soon`, `respawn-ready`) 以套用顏色。
- **頻道按鈕改為 Choice Chips**：record 表單內的頻道按鈕會以 chip 樣式呈現，點選時會帶入 `頻道` 欄位並在視覺上顯示為已選（active）。

如需我將這些變更打成 PR、或是把 `records` 頁面加上排序預設與匯出篩選結果的按鈕，我可以接著實作。
